# ■ アプリ概要

ローカルのMP3ファイルを取り込み、
キーボード操作で再生できるデスクトップアプリ。

配信やローカル用途でのサウンド再生を想定。

========================================
# ■ 機能要件

----------------------------------------
## 1. MP3管理

- ユーザーはMP3ファイルを複数取り込める
- ファイルはアップロードではなく「ローカルパス」で管理
- MP3は一意IDで管理する

### 各MP3の情報
- id: string（UUID）
- name: string
- filePath: string
- duration: number（可能なら取得）
- keybinds: string[]（最大3）
- isPlaying: boolean

----------------------------------------
## 2. 再生制御（重要）

- 同一MP3の同時再生は禁止
- 異なるMP3は並列再生可能
- 最大同時再生数はデフォルト10（設定で変更可能）

### 再生ルール
- キー押下時：
  - 再生中 → 何もしない
  - 停止中 → 再生

- 再生終了時：
  - 自動で再生状態を解除

----------------------------------------
## 3. キーバインド（重要）

- ElectronのglobalShortcutを使用
- アプリが非アクティブでも反応する

### 仕様
- 1つのMP3に最大3つのキーを設定可能
- 同一MP3内で同じキーは登録不可
- 異なるMP3間でキーの重複は許可

### 重複時の挙動
- 警告表示は行うが登録は可能
- キー押下時：
  - 該当キーを持つMP3をすべて取得
  - 各MP3について再生判定を行う

----------------------------------------
## 4. プリセット管理

### タブ構成
- 「全体」タブ（削除不可）
- ユーザー定義プリセット（追加・削除・リネーム可能）

### データ構造
- プリセットはMP3の参照（ID配列）で管理

### 取り込みルール
- 全体タブで追加 → 全体のみ
- プリセットで追加 → 全体 + 該当プリセット

### プリセット操作
- 全体から任意のプリセットへ追加可能
- 重複追加は禁止

----------------------------------------
## 5. 削除仕様（重要）

### A. 完全削除
- MP3を完全に削除
- 全プリセットから削除
- 再生中なら停止

### B. プリセットから削除
- 対象プリセットからのみ削除
- MP3自体は残る

### UI制御
- 全体タブ：完全削除のみ
- プリセットタブ：
  - 完全削除
  - プリセット削除

----------------------------------------
## 6. ランダム再生（重要）

- プリセット単位で実行

### 必須仕様
1. 全MP3をシャッフルしたキューを生成
2. 1周するまで同じMP3を再生しない
3. キューが空になったら再生成（ループ）

- ON/OFFトグルで制御
- OFFにするまで自動再生継続

----------------------------------------
## 7. オーディオ出力

- 再生デバイス選択機能を提供
- 利用可能なデバイス一覧を取得
- 選択デバイスへ出力

----------------------------------------
## 8. 永続化

- JSONファイルで保存

### 保存対象
- MP3情報（パス・キー）
- プリセット構成
- 設定（最大同時再生数など）

----------------------------------------
## 9. UI要件

- タブUI（全体 / プリセット）
- MP3一覧表示
- キーバインド設定UI
- 再生中の視覚表現：
  - ハイライト
  - 再生アイコン
- ランダム再生トグル
- 設定画面

----------------------------------------
# ■ 非機能要件

- 低遅延再生
- 安定した並列再生
- 状態の整合性を保証

----------------------------------------
# ■ 技術要件

- Electron
- React
- globalShortcut使用
- 音声：HTMLAudioElement or Web Audio API
- 状態管理：Context API または Zustand

----------------------------------------
# ■ アーキテクチャ設計（必須）

以下の責務分離を厳守：

## AudioManager（最重要）
- 再生状態の一元管理
- 再生中MP3のMap管理
- 同時再生制御
- 再生終了処理

## ShortcutManager
- globalShortcut登録・解除
- キーバインド変更時の再登録

## PresetManager
- プリセット管理
- MP3参照管理

## StorageManager
- JSON保存・読み込み

----------------------------------------
# ■ エラーハンドリング

- ファイルパス無効時の処理
- 再生失敗時の処理
- キーバインド競合警告
