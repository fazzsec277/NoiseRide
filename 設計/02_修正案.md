### colDrag の動作まとめ

```
表示   状態              クリック後
────   ───────────────   ──────────────────────────────
▶      デフォルト順       再生中ソート（降順）に切り替わる
≡      再生中ソート中     デフォルト順に戻る
≡      他のソート中       デフォルト順に戻る（既存の動作）
```

> **`▶` が表示されるのはデフォルト順のときだけ**。  
> ソートが有効なときは常に `≡`（リセット）が表示される。

現在のキューリスト構造：


playedHistory (スタック):  [A, B, C]
currentPlaying:            D
queue (シャッフル済みFIFO): [E, F, G, ...]
Auto-advance（曲が終わる）: getNext() → queue から先頭を取り出して再生
⏭ 次の曲: getNext() → 同様に queue から新しい曲（futureQueue がないので）
⏮ 前の曲: playedHistory からポップして再生
問題点： 前の曲に戻った後に「次の曲」を押すと、戻る前にいた曲（D）ではなく queue からランダムに新しい曲が取り出される。


⏮ 押す: C を再生、D は消える（futureQueue がない）
⏭ 押す: queue から E が出てくる（本来 D に戻るべき）
修正方法： futureQueue（前進履歴スタック）を追加する。


⏮ 押す: currentId(D) を futureQueue に積む → C を再生
⏭ 押す: futureQueue が空でなければ futureQueue から D を取り出す
         空なら通常通り queue からランダムに取り出す
曲が自然に終わる: futureQueue をクリア（ブラウザの「戻る」と同じ仕様）
