本アプリは以下のコマンドでも実行できますが、exeをたたいて実行する方法を教えて

npm run build:win で Windows 用インストーラー付き exe をビルドできます。手順は以下です。

1. ビルドコマンド（PowerShell）


cd /d "C:\Users\exia0\Documents\01_ClaudeCode\NoiseRide"
set PATH=C:\Program Files\Microsoft Visual Studio\2022\Community_2\MSBuild\Microsoft\VisualStudio\NodeJs;%PATH%
npm run build:win
2. 生成される場所

ビルド完了後、dist\ フォルダに以下が生成されます:


dist\
  NoiseRide Setup 1.0.0.exe   ← インストーラー
  win-unpacked\
    NoiseRide.exe              ← インストール不要の直接実行版
3. 実行方法

方法	ファイル	説明
インストールして使う	NoiseRide Setup 1.0.0.exe	スタートメニューに登録される
そのまま実行	win-unpacked\NoiseRide.exe	インストール不要、フォルダごと移動可能
注意点

ビルドには数分かかります
uiohook-napi（グローバルホットキー）はネイティブモジュールなので、初回ビルド時に Visual Studio Build Tools が使われます（C:\Program Files\Microsoft Visual Studio がすでにある環境なので問題ないはずです）
開発中の変更を反映させるには毎回ビルドし直す必要があるため、動作確認は npm run dev の方が効率的です

現在のキューの動作を説明してから修正案を示します。

現在のキューリスト構造：


playedHistory (スタック):  [A, B, C]
currentPlaying:            D
queue (シャッフル済みFIFO): [E, F, G, ...]
Auto-advance（曲が終わる）: getNext() → queue から先頭を取り出して再生
⏭ 次の曲: getNext() → 同様に queue から新しい曲（futureQueue がないので）
⏮ 前の曲: playedHistory からポップして再生
問題点： 前の曲に戻った後に「次の曲」を押すと、戻る前にいた曲（D）ではなく queue からランダムに新しい曲が取り出される。


⏮ 押す: C を再生、D は消える（futureQueue がない）
⏭ 押す: queue から E が出てくる（本来 D に戻るべき）
修正方法： futureQueue（前進履歴スタック）を追加する。


⏮ 押す: currentId(D) を futureQueue に積む → C を再生
⏭ 押す: futureQueue が空でなければ futureQueue から D を取り出す
         空なら通常通り queue からランダムに取り出す
曲が自然に終わる: futureQueue をクリア（ブラウザの「戻る」と同じ仕様）
